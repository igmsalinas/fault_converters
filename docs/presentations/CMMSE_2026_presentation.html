Context

Why not just use threshold alarms?

The gap

Parametric drift migrates poles and zeros — a scalar alarm cannot see it.

H(s) = C(sI − A)−1B + D

Near its operating point a switching converter is a closed-loop LTI system. Component aging perturbs the state matrices (A, B, C, D), reshaping the small-signal transfer function across decades of frequency.

Reading the full Bode response H(jω) exposes capacitor aging, MOSFET on-resistance wear and ESR growth long before any single-point limit is crossed.

Non-linear coupling. A ±10% ESR change reshapes magnitude and phase across decades — no fixed limit can bound it.

High dimensionality. Each state is a 202-D vector (101 magnitude + 101 phase); per-point limits do not scale.

Label scarcity. Field faults are rare and unlabeled — train on healthy data only, self-supervised.

Edge budget. Real-time inference beside the converter, within a few watts and sub-millisecond latency.

Our answer. Contrastive learning (CARLA) for sensitivity; on-device compilation for edge deployment.

Self-supervised contrastive learning

CARLA separates health from degradation.

09
  • Reconstruction-only autoencoders learn to rebuild normal data — limited discriminative power[9, 10].
  • Point: spikes, dropouts, band-limited noise.
  • Contextual: amplitude/phase offsets, gain and roll-off slope changes.
  • Collective: resonance shift/addition, bandwidth change, oscillation.
  • Physics-based: ESR growth, inductor saturation, Rds increase — matched to real fault modes.
  • Projection head g(·): a 2-layer MLP maps latents to a 64-D unit sphere where NT-Xent (τ = 0.1) operates[1, 4].
  • Inference scores non-parametrically by k-NN distance (k = 5) over the projection space — threshold-independent ROC-AUC.

NT-Xent objective

Maximally separate healthy and degraded states in latent space.

LNT-Xent = − log exp( sim(zi, zj) / τ ) Σk≠i exp( sim(zi, zk) / τ )
zi, zjprojections of a positive (same-state) pair zkall other samples in the batch (negatives) sim(·,·)cosine similarity between projections τtemperature scaling the similarities

Objectives, scoring & search

From reconstruction error to a decision.

10

Reconstruction objective (MSE)

Lrecon = 1N Σi=1N ‖ xi − x̂i ‖2

Per-sample anomaly score

s(x) = 1L·F Σt,f ( xt,f − x̂t,f )2

CARLA score — k-NN distance in latent space

sCARLA(x) = 1k Σj ∈ Nk(x) ‖ z(x) − zj ‖

VAE objective — negative ELBO

LVAE = Lrecon + β DKL( q(z|x) ‖ N(0, I) )

Threshold calibration (normal errors only)

τ = P95 { s(x) : y = normal } τ = μN + 3 σN   (std method)

Decision rule

ŷ(x) = 𝟙[ s(x) > τ ]

Bayesian hyperparameter search

θ* = argmaxθ ∈ Θ AUC(θ) αEI(θ) = 𝔼[ max( f(θ) − f(θ+), 0 ) ]

Reconstruction models score by MSE; CARLA scores by k-NN distance in the projection space. Every threshold is calibrated on healthy data only — no anomaly labels required at deployment.

Evaluation protocol

Ranking first, then the operating point.

11

Operating-point metrics

Precision = TPTP + FP Recall = TPTP + FN F1 = 2 · Precision · RecallPrecision + Recall Accuracy = TP + TNTP + TN + FP + FN Separability = μanom − μnorm1 + σnorm

TP / FP / TN / FN from the calibrated decision ŷ(x) = 𝟙[ s(x) > τ ]. All computed with scikit-learn on the held-out test set.

Threshold-free ranking

ROC-AUC = ∫01 TPR  d(FPR) PR-AUC = ∫01 Precision  d(Recall)

Threshold calibration — healthy data only

  • Percentile — τ = Pp of normal reconstruction error (p = 90–97).
  • Std — τ = μN + 3σN.
  • F1-optimal — grid search maximizing validation F1.
  • Youden J — maximize TPR − FPR on the ROC curve.

AUCs are threshold-independent, so they expose true ranking even when a fixed operating point fails to transfer — the key diagnostic for the quantization results ahead.

Protocol

Batch-1 (n=1) every backend; 15 warm-up + 150 timed runs; wall-clock mean ± std.

Single-sample timing isolates compiler optimisation from batch amortization; classification metrics (accuracy, F1, ROC-AUC) are recomputed in the same pass — speed and fidelity from one measurement.

Conclusion & future work

Sensitive, non-invasive, real-time.

17
  • Modular self-supervised framework (CARLA) for non-invasive converter fault diagnosis.
  • Frequency-response signatures are highly sensitive to subtle parametric degradation.
  • On-device compilation runs the 3.42 MB detector in 0.146 ms at batch-1 — 133× faster than eager execution, with no accuracy loss.
Embedded edge-AI GPU

Port to NVIDIA Jetson Orin (Ampere GPU + DLA): on-device TensorRT with no PCIe host copy, at 5–60 W[15].

FPGA dataflow

hls4ml-style pipelined inference for deterministic sub-microsecond (~100 ns) latency inside the control loop[16].

State space models

Mamba-style linear-complexity sequences, targeting ~8× speedup over Transformers[11].

Physics-informed nets

Embed Buck-converter ODEs into the contrastive loss to generalize with less synthetic data[12].

Power & energy

Extend benchmarks to Watts and Joules-per-inference for thermally constrained systems.

KANs

Kolmogorov-Arnold Networks for symbolic extraction and interpretability.

One reproducible loop — physics-based simulation, self-supervised representation learning and hardware-aware compilation — closes the gap between detection accuracy and real-time edge constraints.
